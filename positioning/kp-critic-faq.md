# KP:1 Critic FAQ (Draft)

## Why not just use JSON-LD, RDF, or plain documents?

KP:1 is focused on epistemic state representation (belief, uncertainty, evidence,
contradiction, and evolution) for AI reasoning workflows. It is not claiming to
replace all semantic web or document standards.

## Is this just a proprietary format?

The intent is open publication with explicit draft status, conformance tests, and
public critique loops. The format should be judged by clarity, implementability,
and results.

## Are you overfitting to one internal system?

This is a valid concern. The mitigation is explicit strict core boundaries,
separate extension profiles, and external implementation attempts.

## Where is the evidence it improves outcomes?

Evidence is currently partial. Benchmark publication is a core roadmap item and a
precondition for stronger adoption claims.

## Why include contradictions instead of resolving them?

Contradictions are often part of real-world knowledge. Preserving and typing them
can improve reasoning quality versus premature flattening.

## Is this a standard?

Not yet. It is a public draft intended to improve through implementation and
challenge.
