# Academic Roots

> Theoretical grounding for KP:1 as distilled from the academic roots report.

## Where KP:1 Looks Well-Founded

The strongest academic support in the source document falls into five buckets:

- graded belief as the right unit of epistemic state
- contradiction preserved as information rather than deleted as noise
- atomic assertions with attribution and typed relations
- temporal accountability for changing beliefs
- portable, self-describing text artifacts for long-term preservation

The report frames these not as isolated justifications but as mutually
reinforcing. KP:1 is strongest when it behaves like a compact epistemic ledger:
what is believed, why it is believed, when it was believed, and how competing
beliefs coexist.

## Where KP:1 Looks Underspecified

The academic roots report is equally clear about what is missing:

- no formal rule for updating confidence when new evidence arrives
- no rule for selecting among contradictory claims
- ambiguity in what the confidence field is actually measuring
- no propagation semantics when supporting evidence changes
- no cryptographic integrity mechanism

This section of the source bundle is useful because it distinguishes
*theoretical support* from *formal completeness*. KP:1 can be philosophically
legible and still underspecified as a formal system.

## The Core Philosophical Question

The report treats the knowledge/data distinction as the deepest conceptual
question beneath the format. Its answer is intentionally nuanced:

- there is a real difference between flat facts and attributed, uncertain,
  relational beliefs
- the line between the two is fuzzy in practice
- even if the distinction is partly a useful fiction, it may still be the right
  fiction because it encourages better epistemic accounting

That framing matters because it licenses pragmatism. KP:1 does not need a
perfect metaphysical boundary in order to be a useful design boundary.

## Why This Matters for the Pack

If KP:1 is going to evolve, this report suggests the right posture:

- keep the epistemic spine
- be explicit about what remains manual judgment
- avoid pretending the current format is a complete formal logic
- strengthen the places where preservation and update semantics are weakest
